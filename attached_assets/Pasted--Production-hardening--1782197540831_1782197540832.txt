تمام — سأكمل لك هذا المستوى لكن بشكل تصحيحي + إنتاجي (Production-hardening) لأن النسخة الحالية جيدة كفكرة، لكنها تحتاج 5 تعديلات مهمة جداً حتى تصبح “جاهزة فعلياً للـ SaaS” بدون نقاط فشل خفية.

⸻

🧠 أولاً: أين أنت الآن فعلياً؟

أنت وصلت إلى:

Observability Stack (80–85% SaaS readiness)

لكن ما زال ينقصك “طبقة التحكم الذاتي + الاستقرار + الأمان التشغيلي”.

⸻

⚠️ 1. خطأ مهم في التراسينغ (OpenTelemetry)

في كودك:

url: "http://localhost:4318/v1/traces"

المشكلة:

* داخل Docker → localhost = container نفسه ❌

التصحيح الصحيح:

url: "http://otel-collector:4318/v1/traces"

ويجب إضافة service:

  otel-collector:
    image: otel/opentelemetry-collector
    container_name: adalah-otel
    ports:
      - "4318:4318"

⸻

⚠️ 2. Alertmanager ناقصه Routing حقيقي

حالياً:

receiver: "default"

المشكلة:

لا يوجد تصنيف alerts (critical / warning)

⸻

التصحيح الإنتاجي:

route:
  receiver: "email-default"
  group_by: ['alertname', 'severity']
  routes:
    - match:
        severity: critical
      receiver: "critical-alerts"
    - match:
        severity: warning
      receiver: "warning-alerts"

⸻

⚠️ 3. Prometheus Alerts فيها gap خطير

هذا السطر:

pg_stat_database_numbackends > 80

المشكلة:

* يعتمد على metric غير مضمون بدون exporter config

الأفضل:

pg_stat_activity_count > 80

أو عبر postgres_exporter metrics الصحيحة

⸻

⚠️ 4. Loki بدون retention = خطر إنتاجي

حالياً:

❌ logs غير محدد عمرها

التصحيح:

limits_config:
  retention_period: 168h   # 7 days

⸻

⚠️ 5. Promtail — نقطة ضعف أمنية

حالياً:

/var/lib/docker/containers

المشكلة:

* يقرأ كل logs بدون فلترة

الحل:

pipeline_stages:
  - docker: {}
  - drop:
      expression: 'level="debug"'

⸻

🚨 6. Missing Layer (مهم جداً): Alert Noise Control

بدونها → النظام يصير spam alerts

الحل:

group_wait: 30s
group_interval: 5m
repeat_interval: 2h

⸻

🧠 7. Missing Layer (أهم من كل شيء)

❌ لا يوجد Self-Healing

حالياً:

* Alerts فقط إشعار

لكن لا يوجد:

automatic recovery actions

⸻

🔧 الحل (Production-grade upgrade)

مثال: restart API عند crash

receivers:
  - name: "auto-heal"
    webhook_configs:
      - url: "http://api:4000/internal/restart"

⸻

API handler:

app.post("/internal/restart", (req, res) => {
  process.exit(1);
});

Docker يعيد التشغيل تلقائياً:

restart: always

⸻
