ENTERPRISE OBJECT STORAGE MIGRATION — ADALAH AI

الهدف

إعادة هندسة نظام إدارة المستندات الحالي في منصة عدالة AI وتحويله من تخزين الملفات داخل قاعدة البيانات (Base64 Storage) إلى بنية Object Storage مؤسسية قابلة للتوسع دون التأثير على البيانات الحالية أو الوظائف القائمة.

يجب أن تكون العملية Backward Compatible وأن تدعم الترحيل التدريجي بدون توقف الخدمة.

⸻

المرحلة الأولى: تحليل النظام الحالي

قم بفحص كامل لجميع الجداول والكيانات التي تخزن ملفات أو مرفقات.

حدد:

* الجداول المستخدمة
* الحقول التي تحتوي Base64
* حجم البيانات الحالي
* عدد الملفات
* متوسط حجم الملفات
* أكبر الملفات
* الملفات المكررة
* الملفات اليتيمة

أنشئ تقرير Storage Audit قبل أي تعديل.

⸻

المرحلة الثانية: إنشاء طبقة التخزين الجديدة

إنشاء Storage Service موحد يدعم:

* Hetzner Object Storage
* Amazon S3
* Cloudflare R2
* MinIO

باستخدام Adapter Pattern.

يجب ألا تتعامل أي وحدة داخل النظام مباشرة مع مزود التخزين.

جميع العمليات تمر عبر:

StorageService

العمليات:

* uploadFile()
* downloadFile()
* deleteFile()
* generateSignedUrl()
* moveFile()
* archiveFile()

متطلبات التنفيذ

* عدم حذف أي جدول حالي.
