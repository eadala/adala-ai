تمام—الآن سننقل “عدالة” من مستوى Dashboard جميل إلى مستوى نظام مؤسسي (Enterprise Design System)، أي نفس البنية التي تستخدمها أنظمة مثل Stripe / Salesforce / Notion من حيث التنظيم، التوسع، وتعدد العملاء.

سأبنيه لك كـ Architecture كاملة + UI System + قواعد هندسية وليس مجرد صفحات.

⸻

🏛️ ما معنى “نظام مؤسسي كامل” فعليًا؟

هو ليس UI فقط، بل 4 طبقات:

1. Design System (Tokens + Components)
2. App Architecture (Multi-tenant structure)
3. Permission System (Roles & Access)
4. Scalability Layer (Pages + Modules)

⸻

🧱 3) Component System (Enterprise Level)

🔹 Base Component Rules

كل Component لازم يكون:

* قابل لإعادة الاستخدام
* لا يحتوي business logic
* قابل للتوسع (props-driven)

⸻

🧩 Card System (Enterprise)

export function Card({ children, variant = "default" }) {
  const styles = {
    default: "bg-white border border-border rounded-md p-4",
    elevated: "bg-white border border-border rounded-md p-4 shadow-sm",
    interactive: "bg-white border border-border rounded-md p-4 hover:shadow-md transition"
  };
  return (
    <div className={styles[variant]}>
      {children}
    </div>
  );
}

⚖️ 4) Case Domain Component (مؤسسي فعلي)

export function CaseCard({ caseItem, onClick }) {
  return (
    <div
      onClick={onClick}
      className="
        bg-white border border-border
        rounded-md p-4 cursor-pointer
        hover:shadow-md transition
      "
    >
      <div className="flex justify-between items-start">
        <div className="text-lg font-semibold">
          قضية #{caseItem.number}
        </div>
        <span className={`
          text-xs px-3 py-1 rounded-full
          ${caseItem.status === "open" ? "bg-green-100 text-green-700" : ""}
          ${caseItem.status === "review" ? "bg-yellow-100 text-yellow-700" : ""}
        `}>
          {caseItem.statusLabel}
        </span>
      </div>
      <div className="text-sm text-gray-500 mt-2">
        {caseItem.court}
      </div>
      <div className="text-xs text-gray-400 mt-1">
        آخر تحديث: {caseItem.updatedAt}
      </div>
    </div>
  );
}

🔐 6) نظام الصلاحيات (Enterprise RBAC)

Roles:

- Super Admin (أنت)
- Office Admin (مكتب محاماة)
- Lawyer (محامي)
- Clerk (موظف)
- Viewer (قراءة فقط)

⸻

مثال حماية UI:

export function Can({ role, allowed, children }) {
  if (!allowed.includes(role)) return null;
  return children;
}

🚀 8) لماذا هذا “Enterprise System” فعلاً؟

✔ ليس مجرد UI

* Multi-tenant architecture
* Role-based access control
* Component-driven system
* Scalable layout structure

* نظام فواتير
* نظام عقود
* Marketplace (محامين)
* API SaaS كامل
