Database Architecture

جميع المكاتب تستخدم قاعدة بيانات PostgreSQL واحدة.

جميع الجداول متعددة العملاء يجب أن تحتوي على:

office_id UUID NOT NULL

JWT Authentication

اعتمد JWT كمصدر وحيد لهوية المستخدم والمكتب.

يجب أن يحتوي JWT على:

{
  "user_id": "...",
  "office_id": "...",
  "role_id": "..."
}

يمنع استخدام:

* query parameters
* request headers
* local storage

لتحديد office_id.

⸻

Tenant Middleware

أنشئ Tenant Middleware مركزي.

المهام:

* التحقق من JWT
* استخراج user_id
* استخراج office_id
* استخراج role_id

ثم إضافة:

req.userId
req.officeId
req.roleId

في جميع الطلبات.

رفض الطلب عند غياب أي قيمة.

⸻

PostgreSQL Row Level Security

فعّل RLS على جميع الجداول متعددة العملاء.

مثال:

ALTER TABLE cases ENABLE ROW LEVEL SECURITY;

إنشاء Policy:

CREATE POLICY tenant_isolation
ON cases
USING (
  office_id =
  current_setting('app.tenant_id')::uuid
);

كرر السياسة لجميع الجداول متعددة العملاء.

⸻

Tenant Context Injection

في بداية كل Request:

SELECT set_config(
  'app.tenant_id',
  $1,
  true
);

بحيث يتم تمرير office_id القادم من JWT.

إنشاء Query Wrapper مركزي.

يمنع أي استعلام مباشر على الجداول متعددة العملاء.

إضافة فحص تلقائي يكتشف:

* SELECT بدون Tenant Context
* UPDATE بدون Tenant Context
