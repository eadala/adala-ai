🧠 ثانياً: التحليل العميق المطلوب

حلل المشروع بالكامل واكتشف:

🔴 الأداء

* بطء تحميل الصفحات
* re-render غير ضروري
* state management سيء
* API calls مكررة أو غير مبررة
* heavy components غير مقسمة

🟠 الهيكل

* ضعف فصل الطبقات (UI / Business / Data)
* coupling عالي بين modules
* ملفات ضخمة يجب تقسيمها

🟡 قاعدة البيانات

* queries غير محسّنة
* missing indexes
* duplicate reads/writes
* lack of caching strategy

🔵 SaaS Multi-Tenant

* هل العزل (tenant isolation) صحيح؟
* هل يوجد تسرب بيانات بين العملاء؟
* صلاحيات (RBAC) غير دقيقة؟

🟣 الأمن

* ضعف في authentication / authorization
* عدم وجود rate limiting
* عدم حماية APIs
* مشاكل XSS / CSRF / Injection
* secrets exposure

🟢 الدفع والفوترة

* ضعف في subscription logic
* عدم دعم billing cycles بشكل صحيح
* مشاكل في webhooks
* عدم تتبع المدفوعات بدقة

📡 التكاملات

* WhatsApp Business Cloud API integration
* notifications queue غير محسّن
* عدم وجود retry/failover logic

⸻

🛠️ ثالثاً: المطلوب تنفيذه

⚡ Performance Optimization

* Code Splitting لكل routes
* Lazy loading كامل للـ pages
* React.memo / useMemo / useCallback
* Debounce لكل search/input
* Virtualization للقوائم الكبيرة
* Image optimization + lazy load
* Reduce bundle size

⸻

🧱 Architecture Refactor

* فصل المشروع إلى:

/frontend
/backend
/services
/shared
/modules

* تطبيق Clean Architecture
* فصل business logic عن UI
* تقليل coupling إلى الحد الأدنى

⸻

🔐 Security Hardening

* JWT + refresh token rotation
* RBAC (Role-Based Access Control)
* Rate limiting لكل API
* Input validation (Zod / Joi)
* SQL injection protection
* Audit logging لكل العمليات الحساسة

⸻

🧾 SaaS Multi-Tenant Fix

* Tenant isolation strict enforcement
* كل query لازم تحتوي tenant_id filter
* منع cross-tenant access نهائيًا
* middleware للتحقق من tenant context

⸻

💰 Billing & Payments

* إصلاح نظام الاشتراكات
* دعم:
    * monthly / yearly plans
    * grace periods
    * failed payment retries
* webhook verification
* logging لكل transaction

⸻

📡 WhatsApp & Notifications

* تحسين message queue (Redis recommended)
* retry mechanism للفشل
* deduplication للأحداث
* real-time event handling
* separation بين notification types
