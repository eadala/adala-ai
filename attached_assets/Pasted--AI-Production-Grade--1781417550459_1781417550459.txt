تمام — سأخصصه لك الآن بشكل مباشر على منصة “عدالة AI” كنظام إنتاجي (Production-Grade) يخدم: إدارة مكاتب المحاماة + القضايا + العقود + المستخدمين متعددين داخل نفس المكتب.

⸻

⚖️ عدالة AI — نظام الصلاحيات الديناميكي (Dynamic Permission System)

🧠 الهدف

تمكين كل مكتب قانوني من:

* إنشاء أدوار مخصصة
* تعديل الصلاحيات بدون مطور
* التحكم الكامل في الوصول حسب القضايا والعملاء
* منع أي تضارب بين UI و Backend
* تطبيق نظام Multi-Tenant + ABAC + RBAC Hybrid

⸻

👥 2) أدوار منصة عدالة AI (Default Roles)

هذه الأدوار تأتي جاهزة لكل مكتب:

Role	الوصف
👑 Owner	مالك المكتب (Full Control)
🧑‍💼 Admin	إدارة تشغيل المكتب
⚖️ Lawyer	محامي يدير قضاياه
🗂️ Clerk	موظف إداري / إدخال بيانات

✔ ويمكن إضافة Roles مخصصة لكل مكتب

🧩 4) Permission Behavior داخل عدالة AI

❌ قاعدة صارمة

أي صلاحية غير معرفة = DENY

⸻

👁️ UI Rule (مهم جداً)

الحالة	السلوك
No Permission	Hide completely
Partial Permission	Show partial UI
Admin/Owner	Full visibility

🧠 5) Policy Engine (داخل عدالة AI)

مثال: القضايا

{
  "resource": "cases",
  "action": "read",
  "condition": {
    "assigned_to": "current_user.id"
  },
  "effect": "allow"
}

⸻

مثال: منع الوصول بعد إغلاق القضية

{
  "resource": "cases",
  "action": "update",
  "condition": {
    "status": "closed"
  },
  "effect": "deny"
}

⸻

⚙️ 6) Decision Flow (عدالة AI)

1. تحديد المستخدم + المكتب (Tenant)
2. جلب Roles
3. جمع Permissions
4. تطبيق Policies (ABAC)
5. تطبيق Overrides
6. إخراج القرار النهائي
