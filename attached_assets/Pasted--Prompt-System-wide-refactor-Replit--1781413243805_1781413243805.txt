المرحلة 3: Performance First Architecture

1. Full Lazy Loading

* كل Center يتم تحميله عند الطلب فقط
* لا تحميل شامل عند الدخول

⸻

2. Route Splitting

كل وحدة:

/legal
/finance
/hr
/ai

تتحمل بشكل مستقل

⸻

3. API Optimization

تقليل الـ payload:

* No over-fetching
* Endpoint segmentation:
    * /cases/summary
    * /cases/timeline
    * /cases/metrics

⸻

4. Smart Caching Layer

* React Query / SWR
* Cache per entity
* Stale-while-revalidate

⸻

5. UI Virtualization

في:

* tables
* logs
* timelines

⸻

المرحلة 4: Dashboard Redesign (Executive Level)

المرحلة 5: AI Integration (On-Demand Only)

AI لا يعمل تلقائيًا.

بل فقط:

* Summarize
* Analyze risk
* Suggest actions
* Detect anomalies

المرحلة 6: UX Simplification Rules

طبق القواعد التالية:

* أي معلومة تحتاج > 3 clicks = فشل UX
* أي صفحة تحتوي > 20 عنصر مرئي = تحتاج إعادة تصميم
* أي feature غير مستخدمة = تُزال أو تُخفى
* أي duplication = يُدمج

⸻

المرحلة 8: Performance Targets

بعد التحسين يجب تحقيق:

* First Load < 2 seconds
* Route change < 300ms
* Lighthouse score > 95
* API reduction > 40%
* UI re-render reduction > 50%
