🧱 1) ERD — Database Design (Multi-Tenant SaaS)

🟦 tenants (العملاء)

id (UUID) PK
name
slug UNIQUE   ← الرابط الثابت /u/{slug}
status (active / suspended)
created_at

⸻

🟨 tenant_settings (إعدادات كل عميل)

id (UUID) PK
tenant_id FK → tenants.id
theme_primary_color
theme_logo_url
services_config (JSON)
payment_enabled (bool)
checkout_account_id
landing_config (JSON)
created_at

⸻

🟩 users (مستخدمين النظام + العملاء)

id (UUID) PK
tenant_id FK → tenants.id
name
email
role (ADMIN / CLIENT / STAFF)
password_hash
created_at

⸻

🟧 services (الخدمات داخل كل عميل)

id (UUID) PK
tenant_id FK
title
description
price
is_active
created_at

⸻

🟥 orders (الطلبات)

id (UUID) PK
tenant_id FK
user_id FK
service_id FK
status (pending / paid / processing / done)
total_amount
created_at

⸻

🟪 payments (الدفع)

id (UUID) PK
order_id FK
tenant_id FK
provider (Checkout.com)
payment_status
payment_intent_id
amount
currency
created_at

⸻

🟫 audit_logs (مهم جداً)

id
tenant_id
action
entity_type
entity_id
metadata (JSON)
created_at

🔄 3) Request Flow النهائي (مهم جداً)

User enters URL
      ↓
Next.js route /u/[slug]
      ↓
API: resolve tenant
      ↓
DB lookup (tenants table)
      ↓
Return tenant config
      ↓
Render dynamic page
      ↓
User creates order
      ↓
Payment via Checkout.com
      ↓
Webhook updates payment table
      ↓
Order marked as PAID

⸻
