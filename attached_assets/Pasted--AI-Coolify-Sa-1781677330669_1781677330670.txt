تمام — الآن بننقلك للمرحلة اللي بعدها مباشرة: تحويل “عدالة AI” من مجرد تشغيل على Coolify إلى نظام SaaS إنتاجي فعلي (Production Grade).

هذه المرحلة هي اللي تفرق بين “موقع شغال” و “منصة تُباع وتتحمل ضغط عملاء”.

⸻

🧠 1. أول مشكلة لازم نعالجها: الـ Multi-Tenancy الصح

حالياً عندك SaaS، يعني:

أكثر من مكتب قانوني داخل نفس النظام

❌ الخطأ الشائع:

خلط بيانات كل المكاتب بدون عزل قوي

✅ الحل الصحيح (Standard SaaS Model):

داخل كل جدول في PostgreSQL لازم:

tenant_id: uuid

⸻

🗄 مثال عملي (Drizzle)

export const cases = pgTable("cases", {
  id: uuid("id").primaryKey(),
  tenantId: uuid("tenant_id").notNull(),
  title: text("title"),
  status: text("status"),
});

⸻

🔐 مهم جداً:

كل Query لازم يكون:

where(eq(cases.tenantId, ctx.tenantId))

⸻

⚡ 2. إضافة Worker System (ضروري للـ AI)

أنت عندك:

* تحليل قضايا
* توليد عقود
* Autopilot
* مستندات

هذه لازم ما تشتغل داخل API مباشرة

⸻

🧱 الحل:

نضيف Service جديد:

worker service

⸻

📦 BullMQ + Redis

مثال Queue:

import { Queue } from "bullmq";
export const aiQueue = new Queue("ai-tasks", {
  connection: { host: "redis", port: 6379 },
});

⸻

🧠 Backend يرسل مهمة:

await aiQueue.add("generate-contract", {
  caseId,
  tenantId,
});

⸻

⚙ Worker يعالج:

import { Worker } from "bullmq";
new Worker("ai-tasks", async job => {
  if (job.name === "generate-contract") {
    // call Gemini / OpenAI
  }
});

⸻
