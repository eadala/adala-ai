Feature Request: Cloud Files Integration (Phase 1)

Objective

إضافة ميزة خفيفة واحترافية داخل منصة عدالة تتيح ربط الملفات السحابية المرتبطة بالعملاء والقضايا والعقود دون إنشاء قسم جديد مستقل أو تغيير هيكل النظام الحالي.

يجب أن تكون الإضافة بسيطة للمستخدم النهائي وقابلة للتوسع مستقبلاً إلى نظام إدارة مستندات متكامل.

⸻

UI Placement

عدم إضافة عنصر جديد في القائمة الجانبية.

بدلاً من ذلك يتم إضافة تبويب جديد باسم:

“الملفات السحابية”

داخل الصفحات التالية فقط:

العميل

البيانات | القضايا | العقود | الفواتير | الملفات السحابية

القضية

التفاصيل | الجلسات | المهام | المستندات | الملفات السحابية

العقد

بيانات العقد | الأطراف | المرفقات | الملفات السحابية

⸻

Supported Providers

المرحلة الأولى:

* Google Drive
* Microsoft OneDrive

يجب تصميم البنية بحيث تسمح مستقبلاً بإضافة:

* SharePoint
* Dropbox
* AWS S3

دون إعادة بناء النظام.

⸻

User Experience

عند فتح تبويب الملفات السحابية يظهر للمستخدم:

عنوان:

الملفات السحابية

وصف:

إدارة وربط الملفات المخزنة على Google Drive أو OneDrive المتعلقة بهذا السجل.

أزرار:

* ربط Google Drive
* ربط OneDrive
* رفع ملف
* مزامنة الملفات

⸻

Integration Logic

يتم ربط حساب التخزين السحابي على مستوى المكتب من خلال الإعدادات.

Settings
→ Integrations
→ Cloud Storage

ويتم استخدام الاتصال نفسه داخل جميع أجزاء النظام.

يمنع طلب تسجيل الدخول لكل قضية أو عميل بشكل منفصل.

⸻

File Association Model

عدم تخزين الملفات داخل جداول القضايا أو العملاء.

يتم إنشاء طبقة ربط مستقلة تسمح بربط الملف بـ:

* العميل
* القضية
* العقد

في نفس الوقت عند الحاجة.

يجب أن يكون الملف مخزناً مرة واحدة فقط.

⸻

AI Integration

عند رفع أي ملف يجب إظهار خيار:

“تحليل الملف بالذكاء الاصطناعي”

ويشمل:

* تلخيص المستند
* استخراج أسماء الأطراف
* استخراج التواريخ
* استخراج المبالغ
* تصنيف نوع المستند
* استخراج الكلمات المفتاحية

ويتم حفظ النتائج داخل السجل المرتبط.

⸻

Security Requirements

Mandatory:

* Tenant Isolation باستخدام office_id
* Access Control
* Audit Logging
* Secure File Access
* Malware Scanning
* Permission Validation

يمنع بشكل كامل ظهور ملفات مكتب لمكتب آخر.

⸻

Performance Requirements

* Lazy Loading للملفات.
* عدم تحميل محتوى الملفات عند فتح الصفحة.
* تحميل البيانات عند الطلب فقط.
* الحفاظ على سرعة المنصة الحالية دون أي تأثير ملحوظ.

⸻

Future Compatibility

يجب بناء هذه الميزة كمرحلة أولى من Smart Documents Architecture.

لكن دون إظهار أي تعقيد إضافي للمستخدم حالياً.

الهدف هو توفير تجربة بسيطة الآن مع بنية تقنية تسمح مستقبلاً بالتوسع إلى:

* Document Management System
* Legal Knowledge Base
* AI Search
* RAG Retrieval
* Smart Evidence Vault

دون الحاجة إلى إعادة تصميم أو إعادة هيكلة المنصة.
