🔴 1) إصلاح الكارثة الأساسية: Multi-Tenant Isolation

✅ Middleware إلزامي (Tenant Guard)

ضع هذا في middleware/requireAuthWithTenant.ts:

import { clerkClient } from "@clerk/express";
export async function requireAuthWithTenant(req, res, next) {
  try {
    const userId = req.auth.userId;
    if (!userId) {
      return res.status(401).json({ error: "Unauthenticated" });
    }
    const user = await clerkClient.users.getUser(userId);
    const officeId =
      user.publicMetadata?.officeId ||
      user.privateMetadata?.officeId;
    if (!officeId) {
      return res.status(403).json({ error: "No tenant assigned" });
    }
    req.userId = userId;
    req.officeId = officeId;
    next();
  } catch (err) {
    return res.status(500).json({ error: "Auth middleware failed" });
  }
}

⸻

🔥 إجبار كل الاستعلامات تكون Tenant-scoped

❌ أي استعلام بدون فلتر هذا = خطأ

db.select().from(cases)

✅ الصحيح:

db.select()
  .from(cases)
  .where(eq(cases.officeId, req.officeId));

⸻

💣 منع الخطأ جذريًا (أفضل حل)

أنشئ helper:

export function tenantWhere(table, officeId) {
  return eq(table.officeId, officeId);
}

واستخدمه في كل مكان.

🔐 Role Guard إضافي

export function requireRole(role) {
  return (req, res, next) => {
    const userRole = req.auth?.sessionClaims?.role;
    if (userRole !== role) {
      return res.status(403).json({ error: "Forbidden" });
    }
    next();
  };
}

🟡 4) إصلاح قاعدة البيانات (Indexes + Performance)

🔥 أضف هذه الـ indexes فورًا

-- Cases
CREATE INDEX IF NOT EXISTS idx_cases_office_id ON cases(office_id);
CREATE INDEX IF NOT EXISTS idx_cases_status ON cases(status);
-- Clients
CREATE INDEX IF NOT EXISTS idx_clients_office_id ON clients(office_id);
-- Documents
CREATE INDEX IF NOT EXISTS idx_documents_case_id ON documents(case_id);
-- Invoices
CREATE INDEX IF NOT EXISTS idx_invoices_office_id ON client_invoices(office_id);
-- Contracts
CREATE INDEX IF NOT EXISTS idx_contracts_office_id ON contracts(office_id);

⸻

🔥 Constraint مهم جدًا (منع تسريب البيانات)

ALTER TABLE cases
ADD CONSTRAINT fk_cases_office
FOREIGN KEY (office_id) REFERENCES offices(id);
