2️⃣ Core System Layers

🟦 1. Edge Layer (Cloudflare)

Responsibilities:

* DNS Routing
* SSL termination
* DDoS protection
* WAF rules
* Rate limiting (first layer)

Rules:

* Block suspicious countries (optional)
* Rate limit /api/*
* Protect admin routes

⸻

🟩 2. Frontend Layer (React + Vite)

Deployment:

* Static build
* Served via CDN (Cloudflare / Nginx / Vercel)

Hardening:

* Role-based UI rendering
* Route guards (RoleRoute)
* No trust in frontend permissions
* API-only security enforcement

⸻

🟨 3. Backend API Layer (Node.js + Express)

Core Responsibilities:

* Authentication (Clerk JWT)
* Tenant Isolation (requireAuthWithTenant)
* RBAC enforcement (requirePermission)
* Business logic
* API validation

⸻

Middleware Stack:

Request →
  Auth Middleware (Clerk)
    ↓
  Tenant Resolver
    ↓
  RBAC Permission Check
    ↓
  Rate Limiter
    ↓
  Route Handler

⸻

Hard Rules:

* ❌ No officeId from client
* ❌ No default tenant fallback
* ❌ No frontend trust
* ✔ All checks backend enforced

⸻

🟪 4. AI Gateway Layer

Components:

* AI Service Router
* Prompt Manager
* Tenant Context Injector
* Permission Filter

⸻

Flow:

User Request
   ↓
RBAC Check
   ↓
Tenant Scope Injection
   ↓
Prompt Builder
   ↓
Gemini / OpenAI
   ↓
Filtered Response

⸻

Hard Restrictions:

* AI cannot access cross-tenant data
* AI must respect role permissions
* AI logs every request (audit)

⸻

🟫 5. Business Logic Layer

Modules:

* Cases
* Clients
* Finance
* HR
* Documents
* Scheduling
* Reports

⸻

Rule:

All modules MUST:

where(tenantId)
+ requirePermission()

⸻

🟧 6. Data Layer

PostgreSQL (Core DB)

Tables MUST include:

* tenant_id (mandatory)
* created_by
* timestamps

⸻

Redis

Used for:

* Sessions cache
* Rate limiting
* AI job queue

⸻

Object Storage

Structure:

/tenants/{tenantId}/cases/
/tenants/{tenantId}/documents/
/tenants/{tenantId}/ai/
/tenants/{tenantId}/exports/

⸻

3️⃣ Security Hardening Map

🔐 Layered Security Model

Cloudflare (WAF)
   ↓
Auth (Clerk)
   ↓
Tenant Isolation
   ↓
RBAC
   ↓
Route-level Permission
   ↓
DB-level Filtering

⸻

🚨 Critical Security Rules

1. NEVER trust client input

* no tenantId from frontend
* no role from frontend
* no officeId headers

⸻

2. Every query MUST include:

WHERE tenant_id = req.tenantId

⸻

3. Every action MUST include:

requirePermission("resource:action")

⸻

4. AI MUST be scoped:

* tenant-aware
* role-aware
* document-restricted

⸻
