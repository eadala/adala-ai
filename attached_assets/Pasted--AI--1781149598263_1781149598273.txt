هذه من أهم أجزاء عدالة AI، لأنك لا تريد أن تكون الباقات مجرد أسعار، بل تريد أن تتحكم فعلياً في خدمات كل مكتب حسب اشتراكه، تماماً مثل HubSpot وClio وShopify.

## المعمارية التي أوصي بها

```text
plans
plan_features
office_subscriptions
offices
```

بحيث:

* الباقة تحدد الخدمات المتاحة.
* المكتب يرث الخدمات من الباقة.
* عند ترقية الاشتراك تتفعل الخدمات تلقائياً.
* عند تخفيض الاشتراك تتعطل الخدمات غير المشمولة.

```ts
export const features = pgTable("features", {

    .primaryKey(),

  }).notNull(),

  name: varchar("name", {
    length: 255
  }).notNull()

```

# دالة التحقق من الخدمة

## shared/utils/hasFeature.ts

```ts
export function hasFeature(

officeFeatures: string[],

featureCode: string

) {

return officeFeatures.includes(
featureCode
);

}
```

---

# الاستخدام

```ts
hasFeature(
office.features,
"ai"
);
```

إذا رجعت:

```ts
true
```

فالمكتب يملك الذكاء الاصطناعي.

## الأفضل مستقبلاً

أن تجعل جميع أجزاء عدالة AI مبنية على:

```ts
hasFeature()
```

وليس على أسماء الباقات.

وبذلك تستطيع:

* إنشاء باقات جديدة.
* تعديل الخدمات.
* إضافة مزايا جديدة.
* منح مزايا خاصة لمكتب معين.
* إنشاء باقات مخصصة Enterprise.

بدون تعديل الكود الأساسي للمنصة، وهو نفس الأسلوب المستخدم في Shopify وStripe وHubSpot.
