🚨 P0 INCIDENT — التنفيذ الإلزامي حتى الإغلاق الكامل

هذا البلاغ يُصنَّف P0 (Critical Production Incident)، ولن يُعتبر مغلقًا إلا بعد تحقق جميع معايير القبول بالأدلة.

من هذه اللحظة يُمنع تمامًا:

* إعادة تنفيذ نفس الإصلاحات السابقة.
* اقتراح حلول مبنية على التخمين.
* إعطاء تقارير تقول “يُفترض أن يعمل” أو “من المتوقع أن يُحل”.

المطلوب هو تنفيذ كامل حتى الوصول للسبب الجذري الحقيقي وإزالته.

⸻

المطلوب

أريد تنفيذ التحقيق من البداية حتى النهاية.

ابدؤوا من أول مرحلة في سلسلة التنفيذ:

1. Source Code
2. Git Commit
3. Build
4. Artifact Generation
5. Deployment
6. Runtime
7. Reverse Proxy
8. Clerk
9. React
10. Browser

لا تتجاوزوا أي مرحلة.

⸻

في كل مرحلة

أريد إثباتًا مباشرًا يشمل:

* PASS أو FAIL
* الدليل الكامل
* Logs
* Headers
* Network
* Bundle
* Runtime Evidence

وليس استنتاجات.

⸻

إذا وُجد FAIL

لا تنتقلوا للمرحلة التالية.

بل:

* حددوا السبب الجذري.
* أصلحوه.
* أعيدوا البناء.
* أعيدوا النشر.
* أعيدوا الاختبار.
* ثم واصلوا.

⸻

يمنع إغلاق المهمة إذا بقي أحد البنود التالية

* شاشة التحميل.
* host_invalid.
* Clerk.isLoaded=false.
* Bundle قديم.
* Build قديم.
* آخر Commit غير منشور.
* أي Asset من إصدار قديم.
* أي Health Check يفشل.
* أي HTTP 500.
* أي Runtime Error.

وجود عنصر واحد فقط مما سبق يعني أن المهمة غير مكتملة.

⸻

بعد كل إصلاح

أعيدوا تنفيذ جميع الاختبارات، وليس الاختبار المتعلق بالتعديل فقط.

يشمل ذلك:

* الصفحة الرئيسية.
* تسجيل الدخول.
* إنشاء Session.
* Dashboard.
* Navigation.
* API.
* Clerk.
* Middleware.
* Proxy.
* Bundle.
* Assets.
* Health Checks.
* Deployment Logs.

⸻

المطلوب في نهاية العمل

لن أقبل عبارة:

تم الإصلاح.

بل أريد تقريرًا يحتوي على:

* Commit SHA المنشور.
* Deployment ID.
* Build Timestamp.
* اسم الـ Bundle الجديد.
* Hash الجديد.
* Last-Modified.
* نتيجة /api/__clerk/v1/environment.
* نتيجة Health Checks.
* نتيجة الصفحة الرئيسية.
* نتيجة تسجيل الدخول.
* نتيجة Dashboard.
* صور شاشة تثبت نجاح التشغيل.
* Console خالية من الأخطاء.
* Network خالٍ من أخطاء Clerk.
* إثبات أن جميع الإصلاحات تعمل على Production وليس محليًا فقط.

⸻

تعليمات تنفيذية

إذا اكتشفتم أن السبب الحقيقي في Replit نفسه (Build Cache، Deployment Infrastructure، Artifact Layer، Runtime، Health Check، أو أي خلل في المنصة)، فلا تلتفوا حول المشكلة.

قوموا بتحديدها بالأدلة، وقدّموا إصلاحًا جذريًا أو بديلًا تقنيًا واضحًا، ولا تستمروا في تكرار محاولات النشر لنفس الكود.

لن تُعتبر هذه المهمة منتهية إلا عندما يعمل الموقع بالكامل على بيئة الإنتاج دون أي شاشة تحميل، ودون أي أخطاء Clerk أو Deployment، مع تقرير إثبات كامل لكل مرحلة من مراحل التنفيذ
