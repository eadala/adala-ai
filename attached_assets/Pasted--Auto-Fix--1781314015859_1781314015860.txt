# 🧩 خطوات التنفيذ الإلزامية:
## 1. تحليل شامل
اقرأ جميع الملفات في المشروع وحدد:
- الأخطاء البرمجية
- التعارضات بين الملفات
- التكرار (Duplication)
- نقاط الضعف الأمنية
- مشاكل الأداء
- تصميم غير صحيح
---
## 2. إصلاح تلقائي (AUTO FIX MODE)
قم مباشرة بـ:
### 🔴 إصلاح الأخطاء:
- إصلاح جميع runtime errors
- إصلاح async/await issues
- إصلاح webhook failures
- إصلاح API routing issues
### 🟠 إصلاح المنطق:
- توحيد منطق Hermes Agent
- منع التكرار في الردود
- إصلاح routing غير الصحيح للرسائل
- منع loops أو double processing
### 🟡 تحسين الكود:
- إعادة كتابة الكود غير النظيف
- إزالة التكرار
- فصل المسؤوليات (Separation of Concerns)
- تحسين naming conventions
---
## 3. Security Hardening
قم تلقائيًا بـ:
- حماية جميع endpoints
- التأكد من صحة التوكنات (Tokens validation)
- منع أي injection risks
- حماية webhooks من التلاعب
- إضافة input validation
- تأمين environment variables usage
---
## 4. Architecture Refactoring
أعد تنظيم المشروع إلى الشكل التالي:
/project
  /core        → منطق Hermes الأساسي
  /routes      → API & Webhooks
  /services    → WhatsApp + External APIs
  /middlewares → Security & Validation
  /config      → Environment & Settings
  /utils       → Helpers
---
## 5. إعادة كتابة الأجزاء المكسورة
إذا وجدت أي ملف:
- غير واضح
- غير آمن
- غير قابل للتوسع
👉 أعد كتابته بالكامل بشكل أفضل (NOT PATCH — FULL REWRITE)
---
## 6. Output إلزامي
بعد الانتهاء، اعرض:
### 🟢 What was fixed
قائمة بكل الإصلاحات التي تمت
### 🔴 Critical issues removed
الأخطاء الخطيرة التي تم القضاء عليها
### 🟡 Improvements applied
التحسينات التي تمت على الأداء والبنية
### 🧠 New architecture summary
شرح سريع للبنية الجديدة
---
