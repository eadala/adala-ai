AI CEO Prompt — Final Enterprise Navigation Upgrade (Justice Platform)

أنت الآن في المرحلة الأخيرة من تطوير معمارية التنقل (Navigation Architecture) لمنصة عدالة.

مهم جداً:
لا تقم بإعادة بناء القائمة الحالية، فقد أصبحت مستقرة. المطلوب هو إضافة طبقة متقدمة من تجربة المستخدم (Enterprise UX Layer) فوق البنية الحالية فقط، مع الحفاظ على جميع الوظائف والروابط والصلاحيات دون أي تغيير.

المبادئ الأساسية

* عدم حذف أي صفحة أو Route.
* عدم تغيير أسماء الوحدات أو منطق الأعمال.
* الحفاظ الكامل على RBAC وFeature Flags.
* عدم التأثير على الأداء.
* عدم كسر أي تكاملات.
* جميع الإضافات يجب أن تكون اختيارية (Progressive Enhancement).
* جميع التعديلات يجب أن تكون Production Ready.

⸻

1. Quick Actions

إضافة قسم ثابت أعلى القائمة بعد مربع البحث بعنوان:

⚡ إجراءات سريعة

يتضمن العمليات الأكثر استخداماً:

* إنشاء قضية
* إنشاء عميل
* إنشاء عقد
* إنشاء مستند
* إنشاء جلسة
* إنشاء مهمة
* إنشاء مراسلة
* إنشاء فاتورة
* تشغيل المساعد القانوني

المتطلبات:

* قابل للتخصيص لكل مستخدم.
* يعتمد على الصلاحيات.
* يعتمد على الباقة.
* يمكن إضافة أو إزالة العناصر.
* يدعم اختصارات لوحة المفاتيح.

⸻

2. Smart Workspace

إضافة مبدل لمساحات العمل (Workspace Switcher).

يوفر أوضاع تشغيل مثل:

* التقاضي
* العقود
* المالية
* الموارد البشرية
* الذكاء الاصطناعي
* الإدارة

عند تغيير مساحة العمل:

* يتم إبراز الأقسام المرتبطة.
* يتم تقليل التشتيت البصري.
* لا يتم إخفاء أي صلاحية.
* لا يتم تغيير أي Route.
* مجرد إعادة تنظيم بصري للقائمة.

⸻

3. Smart Pin

تطوير نظام المفضلة ليشمل:

Pinned Items

يمكن تثبيت:

* قضية
* عميل
* عقد
* مستند
* تقرير
* لوحة

تبقى العناصر المثبتة ظاهرة دائماً أعلى القائمة.

تتم مزامنتها مع حساب المستخدم.

⸻

4. Activity Feed

إضافة لوحة نشاط مصغرة داخل القائمة.

تعرض آخر الأحداث مثل:

* إنشاء قضية.
* تعديل عقد.
* رفع مستند.
* إضافة جلسة.
* دفع فاتورة.
* إشعار جديد.

تعتمد على سجل النشاط الحالي.

لا يتم إنشاء نظام جديد.

⸻

5. Smart Status Indicators

تطوير نظام الـ Badges.

بدلاً من الأرقام فقط:

* عاجل
* يحتاج متابعة
* جديد
* مكتمل
* متأخر
* يحتاج اعتماد

مع استخدام ألوان النظام الحالية.

⸻

6. Advanced Context Navigation

توسيع نظام Context Navigation الحالي.

لا يقتصر على القضايا.

يعمل أيضاً مع:

* العملاء
* العقود
* المستندات
* الموظفين
* الفروع
* الفواتير

عند فتح أي كيان تظهر قائمة سياقية خاصة به.

مثال:

العميل

* نظرة عامة
* القضايا
* العقود
* الفواتير
* المستندات
* المراسلات
* سجل النشاط

بدون إنشاء صفحات جديدة.

⸻

7. Command Palette

إضافة لوحة أوامر احترافية.

الاختصار:

Ctrl + K

أو

⌘ + K

تدعم:

* فتح أي صفحة.
* البحث في القائمة.
* البحث عن القضايا.
* البحث عن العملاء.
* البحث عن العقود.
* البحث عن المستندات.
* تشغيل أوامر النظام.
* تشغيل أدوات الذكاء الاصطناعي.
* إنشاء عناصر جديدة.

تعتمد على الفهرسة الحالية.

⸻

8. Personal Navigation

السماح لكل مستخدم بتخصيص:

* ترتيب الأقسام.
* ترتيب المفضلة.
* ترتيب الإجراءات السريعة.
* طي الأقسام.
* الأقسام المفتوحة افتراضياً.

تتم مزامنة الإعدادات مع حساب المستخدم.

⸻

9. Branch Switcher

إذا كان المكتب يمتلك عدة فروع:

إضافة مبدل فروع احترافي.

المتطلبات:

* تبديل فوري.
* المحافظة على الصلاحيات.
* المحافظة على سياق العمل.
* تحديث البيانات بدون إعادة تحميل كاملة.

⸻

10. Enterprise Accessibility

تحسين تجربة الاستخدام بالكامل:

* دعم كامل للوحة المفاتيح.
* Focus Management.
* Screen Readers.
* WCAG 2.2 AA.
* RTL وLTR.
* High Contrast.
* Reduced Motion.
* Touch Friendly.

⸻

11. Performance

التأكد من:

* Lazy Loading.
* Dynamic Imports.
* Code Splitting.
* Virtual Rendering عند الحاجة.
* عدم زيادة زمن تحميل القائمة.
* عدم زيادة استهلاك الذاكرة.

هدف التنفيذ

الوصول بالقائمة الجانبية لمنصة عدالة إلى مستوى الأنظمة المؤسسية العالمية، بحيث تكون ذكية، قابلة للتخصيص، سريعة، وقابلة للتوسع مستقبلاً، مع الحفاظ على استقرار النظام الحالي بنسبة 100% ودون إدخال أي تعارضات أو أخطاء جانبية.
