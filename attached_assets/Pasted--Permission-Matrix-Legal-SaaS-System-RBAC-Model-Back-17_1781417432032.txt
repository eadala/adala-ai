🔐 Permission Matrix — Legal SaaS System (RBAC Model)

هذا نموذج جاهز لتطبيقه مباشرة في النظام (Backend + Frontend Guards + API Middleware).

⸻

👥 الأدوار (Roles)

* Owner (مالك المكتب)
* Admin (مدير المكتب)
* Lawyer (محامي)
* Clerk (موظف إداري / كاتب)

⸻

🧩 مبدأ التصميم

* Owner = Full Access (System Super Admin داخل نطاق المكتب)
* Admin = إدارة تشغيلية بدون صلاحيات حساسة مالية/ملكية
* Lawyer = تنفيذ قانوني مرتبط بالقضايا فقط
* Clerk = إدخال بيانات + مهام مساندة فقط
* الافتراضي: Deny All → Explicit Allow

⸻

⸻

🔒 Critical RBAC Rules (Must Enforce)

1) Visibility ≠ Disabled

* ❌ ممنوع: عرض عنصر Disabled
* ✅ الصحيح: Hide completely if no permission

⸻

2) Backend Enforcement Mandatory

كل API لازم يتحقق من الصلاحية:

if (!user.hasPermission("cases.read")) {
   return 403
}

⸻

3) Owner Override Rule

if (role == OWNER) {
   grant ALL permissions within tenant scope
}

⸻

4) Scoped Access (Important)

* Lawyer يرى فقط:
    * القضايا الموكلة له
* Clerk يرى فقط:
    * المهام + البيانات المساعدة
* Admin يرى المكتب بالكامل لكن بدون صلاحيات حساسة (مالية / ملكية)
